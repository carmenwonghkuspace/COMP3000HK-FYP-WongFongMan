# COMP3000HK FYP — Email Spam Classification

**Course:** COMP3000HK Computing Project (Final Year Project)  
**Author:** Wong Fong Man  
**Repository:** [carmenwonghkuspace/COMP3000HK-FYP-WongFongMan](https://github.com/carmenwonghkuspace/COMP3000HK-FYP-WongFongMan)

Machine-learning pipeline that classifies email text as **spam** or **ham**, compares several classifiers with cross-validation and statistical tests, exports figures and tables for reporting, and ships a **browser demo** that scores messages locally using exported TF-IDF + logistic weights.

---

## 專案簡介

本專案以 TF-IDF 文字特徵配合多種分類器（Logistic Regression、Random Forest、XGBoost）進行垃圾郵件偵測，並產出交叉驗證結果、配對 t 檢定、SHAP 解釋圖表，以及可在瀏覽器離線推論的 Web 介面。原始電郵資料集體積較大，不會放入 Git；請依照下方說明自行下載至 `datasets/`。

---

## Features

- Text preprocessing (URLs → `url`, numbers → `num`, stop-word removal, uni/bi-grams)
- Model comparison: Logistic Regression, Random Forest, XGBoost
- Stratified 5-fold CV with bootstrap confidence intervals and pairwise t-tests
- Hold-out evaluation, ROC curves, confusion matrix, odds-ratio and SHAP plots
- Serialized model bundle (`result/best_model.pkl`) and browser JSON (`web/model.json`)
- Interactive **Spam likelihood** UI under `web/` (no backend required)

## Dataset setup

The pipeline expects:

```text
datasets/data.csv
```

with at least text and label columns (labels are remapped in `main.py`: class `2` → spam, class `0` → ham; label `1` is dropped).

**To obtain the data:**

1. Configure [Kaggle API / credentials](https://www.kaggle.com/docs/api) if needed.
2. Open and run `data pulling.ipynb`, which downloads  
   [The Biggest Spam/Ham/Phish Email Dataset (~300k)](https://www.kaggle.com/datasets/akshatsharma2/the-biggest-spam-ham-phish-email-dataset-300000)  
   and copies it to `datasets/data.csv`.

The `datasets/` folder is listed in `.gitignore` because the CSV is large.

## Quick start

### 1. Train and export artifacts

From the project root:

```bash
python main.py
```

By default **`FAST_RUN=1`** (faster run, ~minutes): balanced subsample of 1,200 ham + 1,200 spam per class. For a heavier, paper-style budget:

```bash
# Windows PowerShell
$env:FAST_RUN="0"; python main.py

# Linux / macOS
FAST_RUN=0 python main.py
```

Outputs are written under `result/` (figures, CSV tables, `.pkl` models) and `web/model.json`.

### 2. Run the web demo

Browsers block `fetch()` on local files opened as `file://`. Serve the `web/` folder over HTTP:

```bash
cd web
python -m http.server 8765
```

Open [http://127.0.0.1:8765](http://127.0.0.1:8765), paste email text, and click **Score & explain**.

### 3. Regenerate browser JSON only

If you already have `result/best_model.pkl` and do not want to rerun the full pipeline:

```bash
python export_browser_json_from_pkl.py
```

If the winning model is not logistic regression, this script may need `result/processed_tfidf.csv` from the same training run (see script docstring).

## Training pipeline (`main.py`)

| Stage | Output (examples) |
|--------|-------------------|
| EDA | `result/fig1_class_distribution.png` |
| TF-IDF matrix | `result/processed_tfidf.csv` (gitignored), vectorizer `.pkl` |
| CV comparison | `result/table1_cv_results.csv`, `table1b_pairwise_ttest.csv`, `fig2_model_comparison.png` |
| Hold-out test | `table6_holdout_results.csv`, `fig3`–`fig5` |
| Interpretability | `table2`–`table5`, `fig6`–`fig8` (SHAP) |
| Models | `result/best_model.pkl`, `best_model_only.pkl` |
| Browser pack | `web/model.json`, `result/browser_model.json` |

The browser uses a **logistic regression** representation: either the CV-winning model (if it is `LogisticRegression`) or an LR surrogate trained on the same TF-IDF features.
